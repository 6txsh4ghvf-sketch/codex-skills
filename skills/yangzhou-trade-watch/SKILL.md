# yangzhou-trade-watch

## 1. Skill 名称

name: yangzhou-trade-watch

## 2. Skill 用途

本 Skill 用于将公开国际经贸动态整理为服务扬州外贸分析专报的信息卡片、月度观察和专报段落草稿，重点关注来源核验、扬州关联判断和专报转化。

本 Skill 的核心任务不是泛泛收集新闻，而是把外部信息从“新闻线索”加工成“可核验、可筛选、可转化的专报素材”。

主要服务以下工作：

1. 国际经贸动态监测；
2. 外部政策变化跟踪；
3. 重点市场、重点产业、贸易规则和物流能源风险研判；
4. 公开信息来源核验；
5. 信息卡片生成；
6. 月度观察整理；
7. 扬州外贸专报段落草稿生成；
8. “值得关注的情况及建议”素材转化。

## 3. 适用场景

当用户提出以下需求时，应考虑使用本 Skill：

1. 整理国际经贸动态；
2. 核验某条外贸相关信息的来源；
3. 生成外贸动态信息卡片；
4. 筛选月度外部形势重点；
5. 分析美国、欧盟、中东、墨西哥、非洲、东盟等市场变化；
6. 分析船舶、光伏、平板显示模组、机械设备、化工品、农食产品等产业变化；
7. 分析关税、贸易救济、出口管制、CBAM、REACH、RoHS、TBT、SPS、USMCA、RCEP等规则变化；
8. 分析红海局势、霍尔木兹海峡、油价、海运费、战争险、港口拥堵等物流能源风险；
9. 将公开信息转化为扬州外贸专报素材；
10. 生成“值得关注的情况及建议”段落草稿。

## 4. 核心原则

1. 不得编造新闻、来源、链接、数据、发布时间和机构判断；
2. 未核实信息不得写成确定性判断；
3. 媒体报道原则上只作线索，应优先追溯官方、国际组织、行业协会、研究机构等原始来源；
4. 来源清单是优先参考，不是封闭白名单；
5. 所有动态必须判断是否能落到江苏、扬州外贸；
6. 无扬州关联、无可靠来源、无现实建议落点的信息，不作为专报重点；
7. 专报建议必须面向市政府层面，不能写成海关自我工作安排；
8. 输出内容不是最终上报稿，必须人工复核；
9. 对没有本地数据支撑的内容，必须标注“需匹配本地数据后再使用”；
10. 涉及真实专报写作时，应提示用户匹配最新扬州本地外贸数据、重点企业情况和公开来源。

## 5. 联网使用规则

本 Skill 分为两种工作模式。

### 5.1 离线整理模式

适用场景：

1. 创建模板；
2. 整理用户手动提供的公开信息；
3. 基于已有信息卡片生成月度观察；
4. 基于已有素材生成专报段落草稿；
5. 检查卡片字段是否完整。

执行规则：

1. 默认不联网；
2. 不自行补充新闻事实；
3. 只基于用户提供的信息、当前项目文件和已有卡片处理；
4. 对不确定内容标注“待核实”；
5. 不把用户提供但未经核验的信息写成确定性判断。

### 5.2 联网核验模式

适用场景：

当用户明确提出“检索”“核验”“监测”“查找最新信息”“搜索公开来源”“联网确认”等要求时，可以使用 Browser、Chrome 或联网能力访问公开网站。

执行规则：

1. 只能访问公开网站；
2. 优先访问官方机构、国际组织、行业协会、专业研究机构、主流数据库等权威来源；
3. 新闻媒体只作线索，涉及政策、数据、机构判断时应追溯原始来源；
4. 不得访问单位内部系统、非公开平台、个人邮箱、网盘、涉密资料或敏感资料；
5. 每条联网获取的信息必须记录来源、发布时间、链接或出处说明、来源等级、核验结论和待核实事项；
6. 若无法确认原始来源，应标注“待核实”或“仅作线索参考”；
7. 不得为了凑数量而使用低质量来源。

## 6. 来源等级

A级：官方机构、国际组织、法定公告、权威统计发布。  
适用方式：可作为专报主要支撑。

B级：行业协会、专业研究机构、主流数据库、头部咨询机构。  
适用方式：可作为产业趋势、市场判断、价格周期、物流风险等方面的重要支撑。

C级：主流财经媒体、专业媒体、企业公告。  
适用方式：可作为线索或背景。涉及政策、数据和机构判断时，应尽量追溯原始来源。

D级：普通转载、自媒体、论坛、无法核实来源内容。  
适用方式：不得直接作为专报判断依据。最多作为线索，必须进一步核验。

明确要求：

1. A级、B级来源优先；
2. C级来源需谨慎使用；
3. D级来源不得直接转化为专报判断；
4. 无发布时间、无原始出处、无数据口径说明的信息，应标注“待核实”。

## 7. 重点关注方向

### 7.1 重点市场

美国、欧盟、东盟、日本韩国、墨西哥、中东、非洲、印度、拉美、英国、澳大利亚等。

### 7.2 重点产业

船舶、光伏、平板显示模组、电子信息、机械设备、农机、汽车零部件、交通装备、化工品、医药材、农食产品。

### 7.3 重点规则和风险

关税、贸易救济、出口管制、TBT、SPS、CBAM、REACH、RoHS、USMCA、RCEP、原产地规则、红海局势、霍尔木兹海峡、油价、海运费、战争险、港口拥堵。

### 7.4 扬州关注对象

船舶、太阳能电池、平板显示模组、基本有机化学品、农业机械、医药材、机械设备、汽车零部件、玩具、塑料制品、机床、进口平板显示模组、进口催化剂、进口电子纸母片、资源类产品、农食产品。

## 8. 检索和判断方法

检索时优先使用“市场 + 产业 + 规则 + 扬州关注对象”的组合方式。

示例：

1. 美国 + 关税 + 机械设备 + 中国出口；
2. 欧盟 + CBAM + 化工品 + 碳规则；
3. 墨西哥 + USMCA + 机床 + 近岸化；
4. 中东 + 红海局势 + 海运费 + 机械设备；
5. WTO + 全球贸易展望 + 货物贸易；
6. Drewry + 集装箱运价 + 亚欧航线；
7. 光伏 + 组件价格 + 海外需求 + 贸易救济；
8. 船舶 + 全球新船订单 + 绿色船舶。

检索结果不得直接写入专报，必须经过：

1. 来源核验；
2. 事实摘录；
3. 扬州关联判断；
4. 是否适合写入专报判断；
5. 专报转化。

## 9. 工作流一：来源核验

输入：一条公开信息、新闻线索、机构报告、政策文件或用户提供的材料。

输出：

1. 来源名称；
2. 来源类型；
3. 来源等级；
4. 发布时间；
5. 链接或出处说明；
6. 是否为原始来源；
7. 是否可公开检索；
8. 是否需要交叉验证；
9. 是否存在口径不明；
10. 是否存在媒体转述、商业推广或观点倾向；
11. 是否可用于专报；
12. 待核实事项。

判断规则：

1. 官方和国际组织优先；
2. 行业协会和专业研究机构次之；
3. 媒体只作线索；
4. 无法核实的内容不得作为确定性判断。

## 10. 工作流二：信息卡片生成

输入：用户手动提供的公开信息，或联网核验后确认的公开信息。

输出：五段式信息卡片。

信息卡片结构：

### 第一段：基础信息

- 动态标题；
- 信息来源；
- 来源链接或出处说明；
- 发布时间；
- 信息获取时间；
- 来源等级；
- 涉及国家或地区；
- 涉及产业；
- 涉及政策、规则或风险；
- 对应关键词；
- 信息卡片编号。

### 第二段：事实摘录

- 原文关键信息摘录；
- 核心事实摘要；
- 涉及数据；
- 涉及机构判断；
- 涉及企业或行业案例；
- 是否为官方发布；
- 是否为媒体报道；
- 是否需要进一步追溯原始来源。

事实摘录部分只记录信息本身，不作延伸判断。

### 第三段：来源核验

- 来源等级：A/B/C/D；
- 是否有原始出处；
- 是否可公开检索；
- 是否有多个来源交叉印证；
- 是否存在口径不明；
- 是否存在时间滞后；
- 是否存在商业推广、观点倾向或二手转述；
- 核验结论。

### 第四段：扬州关联判断

- 对全球贸易形势的影响；
- 对中国外贸的可能影响；
- 对江苏外贸的可能影响；
- 对扬州外贸的可能影响；
- 关联扬州重点产品；
- 关联扬州重点市场；
- 关联扬州重点企业类型；
- 可能影响方向；
- 影响强度；
- 影响时点；
- 是否适合写入月度专报；
- 适合写入专报的位置。

影响方向可选：需求增加、需求减弱、成本上升、物流受阻、合规压力增加、订单转移、市场替代、价格下行、进口机会、出口风险、政策窗口。

影响强度可选：高、中、低、待观察。

影响时点可选：当月可能影响、未来1-3个月可能影响、上半年可能影响、全年可能影响、中长期影响、暂不明确。

### 第五段：专报转化

- 一句话判断；
- 可写入专报的分析段落；
- 可用于标题的小标题；
- 可配套使用的本地数据；
- 可配套调研的企业方向；
- 可形成的建议落点；
- 待核实事项；
- 暂不建议写入的原因。

专报转化部分不能泛泛写国际新闻，必须落到“数据说明什么、风险在哪里、建议落在哪里”。

## 11. 工作流三：月度筛选

输入：一个月内若干信息卡片。

输出：

1. A类：建议写入本月专报；
2. B类：可作背景储备；
3. C类：持续跟踪；
4. D类：暂不采用。

筛选标准：

1. 来源是否可靠；
2. 是否为近一个月内动态；
3. 是否涉及扬州重点市场；
4. 是否涉及扬州重点产品；
5. 是否影响订单、价格、物流、合规、进口或出口走势；
6. 是否能匹配本地数据；
7. 是否有企业调研或案例补充空间；
8. 是否适合写入市政府专报；
9. 是否存在事实不确定或表述风险；
10. 是否有现实可行的建议落点。

月度观察应包括：

1. 总体判断；
2. 重点市场动态；
3. 重点产业动态；
4. 贸易规则和物流能源风险；
5. 对扬州外贸的可能影响；
6. 建议写入专报的重点素材；
7. 建议持续跟踪事项。

## 12. 工作流四：专报段落转化

输入：信息卡片、月度观察素材或用户指定的公开信息。

输出：

1. 小标题；
2. 一句话判断；
3. 权威来源支撑；
4. 扬州数据或产业关联；
5. 企业或产品影响；
6. 面向市政府的建议落点；
7. 可直接修改使用的专报段落草稿。

专报段落必须遵循以下逻辑：

1. 先给总体判断；
2. 再用权威来源支撑；
3. 再落到扬州数据、产品、市场或企业；
4. 再说明可能影响；
5. 最后提出面向市政府的现实可行建议。

写作要求：

1. 不写成新闻综述；
2. 不堆砌国际动态；
3. 不写空泛判断；
4. 不夸大影响；
5. 不把建议写成海关自身工作；
6. 语言克制、直白、清晰；
7. 尽量形成“问题-影响-建议”的闭环；
8. 没有本地数据支撑的内容，应标注“需匹配本地数据后再使用”。

## 13. 文件输出规则

如果在 global-trade-watch 项目中使用本 Skill，应遵循以下输出规则：

1. 信息卡片输出到：`03_information_cards/YYYY-MM/`
2. 月度观察输出到：`04_monthly_briefs/`
3. 可写入专报的段落输出到：`07_outputs/paragraphs_for_report/`
4. 来源表和筛选表输出到：`07_outputs/source_tables/`
5. 简报材料输出到：`07_outputs/briefing_notes/`
6. `07_outputs` 中的内容不是最终上报稿，仍需人工复核事实、数据、口径和表述。

## 14. 禁止事项

1. 禁止编造来源、链接、数据和机构观点；
2. 禁止将未经核实的媒体报道直接转化为专报判断；
3. 禁止把与扬州无关的国际新闻强行写入专报；
4. 禁止提出脱离市政府职责、难以落地的建议；
5. 禁止处理涉密、内部敏感、未公开企业明细数据；
6. 禁止联网检索时访问单位内部系统或非公开平台；
7. 禁止访问个人邮箱、网盘、聊天记录等非必要私域信息；
8. 禁止将项目输出视为最终上报稿；
9. 禁止为了提高信息量而使用低质量来源；
10. 禁止把“可能影响”写成“已经发生”的确定性结论。

## 15. 用户沟通规则

当任务范围不清时，应先说明理解和执行边界。

当信息不足时，应标注“待补充”或“待核实”，不要自行补全。

当用户要求联网检索时，应先明确：

1. 检索时间范围；
2. 重点市场；
3. 重点产业；
4. 是否只使用公开来源；
5. 是否需要生成信息卡片；
6. 是否需要生成专报段落草稿。

当完成任务后，应说明：

1. 创建或修改了哪些文件；
2. 主要判断是什么；
3. 哪些信息仍需核实；
4. 哪些内容可进入月度观察；
5. 哪些内容可转化为专报段落；
6. 哪些内容暂不建议采用。
