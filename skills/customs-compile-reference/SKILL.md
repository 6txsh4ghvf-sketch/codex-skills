# customs-compile-reference

本文件用途：定义 customs-compile-reference Skill 初稿。

## Skill 目的

当用户围绕《海关编译参考》型稿件进行外文材料筛选、选题判断、编译价值评估、文章框架生成、摘译重点提炼、启示建议转化或风格审查时，使用本 Skill。

本 Skill 只输出“候选评估、编译准备稿、框架草稿、摘译重点、审查建议”。不直接输出正式投稿终稿。

## 何时触发

- 用户提供外文材料标题、摘要、链接、PDF 或正文，要求判断是否适合《海关编译参考》。
- 用户要求对候选材料评分、识别稿型、生成文章框架或编译准备稿。
- 用户要求从外文材料中提取核心观点、数据、案例、规则变化或摘译重点。
- 用户要求按《海关编译参考》样稿风格审查标题、文章要点、正文结构或启示建议。
- 用户要求根据白名单来源或主题关键词自主发现候选材料。

## 输入材料要求

优先要求提供：发布机构、发布时间、原文标题、原文链接、材料类型、PDF 或正文文本。若缺失，应标注“待核验”，不得自行补写。

不接受涉密资料、内部系统数据、未公开业务数据、未公开企业明细、执法敏感信息和个人隐私信息。

## 自主来源发现能力

本 Skill 支持根据 `02_sources/source_watchlist.yaml` 和 `02_sources/domain_allowlist.md` 发现候选材料。发现方式包括：

- 白名单来源扫描。
- 主题关键词定向扫描。
- 重点机构扫描。
- 在人工允许时进行补充开放搜索，但结果必须回到官方机构、国际组织、国外海关、权威智库或专业机构原文页面。

自主发现时不得全网无边界乱搜，不得将普通新闻作为主来源，不得编造来源、链接、发布时间、报告名称或外文原文。

## 选题评分流程

1. 检查是否公开来源、是否触及保密边界。
2. 判断来源层级和原文可得性。
3. 判断主题是否属于目录高频方向。
4. 判断海关关联度，重点看监管、税收、通关、检验检疫、口岸安全、风险防控、贸易便利化、国际合作。
5. 按 `04_selection_rules/scoring_rubric.md` 评分。
6. 输出推荐等级：重点候选、一般候选、暂存或不建议写。

自主发现候选材料时，每条候选必须记录外文标题、中文暂译题、发布机构、发布时间、来源链接、材料类型、是否有 PDF、主题标签、推荐稿型和待核验事项。

## 稿型识别流程

优先识别六类核心稿型：

1. 国外海关技术应用类。
2. AI 技术场景完整演示类。
3. 多案例治理能力综合类。
4. 国际海关规则框架解读类。
5. 智库经贸趋势研判类。
6. 国际规则争议与应对类。

必要时使用预留稿型：关键矿产与供应链安全类、检验检疫与生物安全类、跨境电商监管类、出口管制和贸易救济类。无法匹配稿型时，应列入观察，不强行成稿。

## 文章框架生成流程

1. 根据稿型选择 `05_article_templates` 中对应模板。
2. 生成文章要点、正文一级标题、二级要点和人工核验事项。
3. 输出为“编译准备稿”或“框架草稿”，控制在研究辅助范围内。
4. 不写正式投稿终稿，不补写无法核验的来源、数据和原文。

## 摘译重点提取流程

1. 标出适合摘译的核心观点、关键数据、典型案例、规则变化和政策动作。
2. 区分“可摘译”“仅作背景”“待核验”。
3. 保留原文标题、段落位置或页码信息；没有页码时说明依据来自用户提供文本。
4. 不做脱离原文的扩写。

## 风格审查流程

审查标题是否贴近目录风格，文章要点是否为决策提示，正文是否为摘译重组，启示建议是否稳妥且在海关职责边界内。重点排查普通新闻拼接、论文腔、宣传化表达、地方外贸专报口径和脱离原文自由发挥。

## 输出格式

- 候选材料卡片。
- 100 分制评分表。
- 推荐稿型和理由。
- 文章要点和正文框架。
- 摘译重点和待核验事项。
- 风格审查意见和修改建议。

所有输出均须标注“草稿、候选、待人工核验”。

候选材料卡片应优先使用 `07_candidates/candidate_cards/candidate_card_template.md` 的字段。

## 安全边界

只处理公开资料、公开外文报告、公开政策文本、公开新闻稿、公开研究报告和用户明确允许处理的个人练习材料。所有内容默认为内部研究辅助草稿，不代表正式意见。

## 禁止事项

- 不得编造来源、链接、发布时间、发布机构、数据和外文原文。
- 不得处理涉密、内部系统、未公开业务数据、执法敏感信息和个人隐私信息。
- 不得直接生成正式投稿终稿。
- 不得把普通新闻简单拼接成稿。
- 不得将普通新闻作为主来源。
- 不得跳过人工核验。
- 不得把 customs-compile-watch 与 global-trade-watch 混用：前者服务《海关编译参考》型稿件，后者服务扬州外贸分析专报。

## 示例任务

- 评估一份 WCO 报告是否适合《海关编译参考》。
- 为一份 OECD TBT 报告生成候选评分表。
- 将一份国外海关 AI 应用报告转化为编译准备稿框架。
- 按样稿风格审查一份启示建议是否越界。
